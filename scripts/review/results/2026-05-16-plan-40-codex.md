1. **License triage NOT fail-closed for the mixed 30,499-PDF corpus.** The plan's workflow for distinguishing public-domain / CC-licensed sources from vendor-proprietary PDFs depends on operator vigilance during ingest. For an MIT+CC-BY-4.0 public repo where every landed page is potentially indexed/forked/archived in perpetuity, the workflow should be safe-by-construction (deny-list by-default, explicit allow-list per source-family) rather than operator-vigilance dependent. Material license-tainting risk if a single proprietary PDF gets ingested as a wiki page.

2. **Founding event has no rollback / taint-response story.** This is the 11th wiki domain founding. Plan does not specify what happens if a license-tainted page lands and is detected post-publication: revert procedure, downstream-archive notification protocol, fork-cleanup outreach, etc. For a founding event with high reversibility cost, the rollback procedure should be part of the plan, not an afterthought.

4. **Earlier 5-wave PR strategy dropped from canonical plan.** The 2026-05-07 issue comment specified a 5-wave PR landing strategy. The current canonical plan body says "single-issue scope" with no sub-issues and no waves. The wave-PR strategy is either implicit-but-unstated (defect: should be stated) or dead (defect: should be explicitly marked dead-and-why). Either way, this is a plan-vs-prior-state divergence that needs resolution.

5. **Standards anchors too generic for the formation-eval scope.** The current plan names SEG-Y and logging-API specs at the general level. Formation-evaluation has specific recommended-practice and methodology anchors (LAS file format, SPE-formatted reservoir-characterization workflows, SPWLA petrophysics references) that should be enumerated at plan-time, not deferred to drafting-time.
