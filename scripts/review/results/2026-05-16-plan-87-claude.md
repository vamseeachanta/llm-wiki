## Findings

### MINOR-1 — DE/PE well-integrity scope-edge needs to be an acceptance-criteria check, not just risk-table item

**Location:** plan lines 169-171 (Sub-issue 3 router page scope-boundary statement) + plan lines 199-200 (risk mitigation "explicit scope-edge statement").

**Issue:** The plan correctly identifies the drilling-engineering well-construction-integrity vs production-engineering well-integrity-during-production scope-edge as a risk (lines 199-200). It also names the bidirectional cross-link requirement in acceptance criteria (line 216-217). But the substantive REQUIREMENT — that `concepts/well-integrity-during-production.md` open with an explicit scope-edge sentence stating "this page covers operating-time degradation; well-construction integrity is at drilling-engineering/concepts/pa-barrier-philosophy.md" — is not in the acceptance-criteria check list as a separable item, only implicit in "bidirectional cross-links". A future implementer could land the cross-links without the scope-edge prose statement and pass acceptance.

**Evidence:** Phase 3 plan acceptance criteria (`docs/plans/2026-05-15-issue-74-pe-phase-3-stimulation.md` line 135) uses "Bidirectional cross-links: Phase 2 completions ↔ Phase 3 stimulation" as a single bullet without separable scope-edge prose check. Phase 3 was a within-PE-domain epic where scope-edge framing was lower-risk. Phase 4 has a CROSS-DOMAIN scope-edge (PE↔DE) which is materially higher risk for collision and warrants separable check.

**Recommendation:** Add discrete acceptance-criteria bullet: "PE Phase 4 `concepts/well-integrity-during-production.md` opens with explicit scope-edge statement referencing drilling-engineering `pa-barrier-philosophy.md` + `cement-evaluation.md` + `primary-cementing.md`; reciprocal scope-edge note added to those DE pages."

### MINOR-5 — `scale-paraffin-asphaltene-monitoring.md` page risks redundancy

**Location:** plan line 174 (Sub-issue 3 page list — `concepts/scale-paraffin-asphaltene-monitoring.md`).

**Issue:** This page sits between three Sub-issue-1 deposition-family pages (paraffin-deposition.md, asphaltene-precipitation.md, mineral-scale.md — plan lines 117-119) and the Sub-issue-3 integrity-monitoring.md page (plan line 173). Risk: redundant coverage. Each deposition-family page should already cover monitoring instrumentation for its specific deposit (since deposition-management literature treats detection inseparably from prevention/remediation). And `integrity-monitoring.md` already covers the corrosion-coupon / wall-thickness / casing-pressure monitoring infrastructure.

**Evidence:** Plan line 174 frames the page as "operational monitoring instrumentation for the three chemical-deposition families" — which is exactly what the three Sub-issue-1 deposition-family pages should already cover in their respective "monitoring" sections. Risk of 3 separate pages with parallel "Monitoring" sections + 1 integrating page = 4 redundant slices of the same content.

**Recommendation:** User-decision gate: (a) drop `scale-paraffin-asphaltene-monitoring.md` and require each Sub-issue-1 deposition-family page to have a complete "Monitoring" section, OR (b) keep `scale-paraffin-asphaltene-monitoring.md` and require the Sub-issue-1 deposition-family pages to STUB their monitoring sections with cross-link to the integrating page. Recommend (a) — fewer pages, fewer cross-link maintenance points; aligns with Phase 3 precedent where deposition-family pages were not split off from intervention-management context.
