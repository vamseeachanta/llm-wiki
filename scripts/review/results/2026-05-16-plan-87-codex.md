## NEW non-overlapping defects (Top 5 — per `feedback_cross_provider_review_payoff`)

### MAJOR-1: Vendor-IP guard is not testable

The plan forbids "recipes/datasheets" but the acceptance criteria and TDD test list do NOT ban commercial inhibitor product names, dosage ranges, performance curves, lab-screening matrices, or LDHI/KHI/AA package behavior. Generic "no proprietary content" framing will silently pass many leaks. The Phase 3 vendor-archetype framing precedent (Halliburton SmartWell / Schlumberger StimFRAC named-only) is qualitative; for production chemistry (Nalco / Champion-X / Halliburton MultiChem) the proprietary risk surface is much larger and needs enumerated deny-list patterns.

**Recommended fix:** add a pytest scanner that flags commercial inhibitor product names + dosage-rate citations + performance-curve transcriptions. Provide allowed/blocked examples in the public-safety section of the plan.

### MAJOR-2: Calc-citation contract is half-activated

The plan proposes wiki-side `citations:` frontmatter "even if no calc module consumes them," while the actual contract (`workspace-hub/.claude/rules/calc-citation-contract.md` + #2481 + #2685) specifies sidecar emission per calc return and fail-closed at calc time. This creates false compliance — wiki pages carry citation metadata but no executable enforcement layer exists.

**Recommended fix:** either commit to real calc-module activation (add calc-citation entries to digitalmodel for V_e, corrosion-rate, scale-SI, paraffin-WAT, hydrate-stability) OR explicitly mark the citations as documentation-only metadata distinct from the calc-citation contract. The current ambiguous middle position is the silent-failure trap.

### MAJOR-3: Multiphase-flow scope risk is deeper than page length

Claude r1 flagged scope-balloon via page-length bound (≤350 lines). Codex finds that the actual risk is correlation-formula drift: the plan allows formulas + "when each applies" framing for Beggs-Brill, Hagedorn-Brown, Duns-Ros, Gray. Once those formulas land at concept level with operational selection logic, the page reads as implementation-grade calculation guidance — which the wiki firewall explicitly forbids (vs simulator algorithms like OLGA/LedaFlow/PIPESIM which are properly excluded). Page-length bound alone doesn't enforce the prose-vs-implementation boundary.

**Recommended fix:** bound multiphase-flow content to prose-only correlation descriptions with explicit "consult [simulator name] for calculation" pointers. Add a TDD test that flags formula transcription patterns (`= ...` with engineering units) on PE wiki pages.

### MAJOR-4: PE-vs-DE boundary requires cross-wiki edits without ownership control

The plan's acceptance criteria require reciprocal drilling-engineering hand-off notes (boundary-edge prose between PE Phase 4 well-integrity-during-production and DE well-construction integrity). But the plan does NOT specify a separate gate or test for modifying drilling-engineering pages, nor a check that prevents PE production-integrity prose from contaminating DE barrier-establishment pages. The "explicit boundary" Claude r1 noted is enforceable as prose, not as a test.

**Recommended fix:** add a TDD test that grepcheckns DE pages for PE-specific terminology (corrosion-monitoring intervals, scale-deposition prediction, paraffin-WAT) and fails build if found. Or scope-fence the cross-wiki edits to a dedicated PR with separate review.

### MAJOR-5: Silent-failure path in vendor citation policy

The plan's policy "cite vendor by name + general chemistry class only" conflicts with "vendor production-chemistry datasheets forbidden" — the two rules don't have a clear boundary. Without an allowed/blocked example table, implementers may cite vendor marketing pages (which are off-repo-routed per the 2026-05-14 service-provider-data-routing matrix) for LDHI/KHI, paraffin, scale, or corrosion inhibitors and think they complied.

**Recommended fix:** add an allowed/blocked example table to the public-safety section. Examples: ALLOWED `Champion-X markets the XS-7000 series of paraffin inhibitors (polyacrylate-based)`; BLOCKED `Champion-X XS-7000 achieves 12% wax-inhibition at 50 ppm per [vendor datasheet]`.
