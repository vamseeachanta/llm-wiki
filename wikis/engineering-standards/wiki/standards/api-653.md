# API 653 — Tank Inspection, Repair, Alteration, and Reconstruction

> **Standard identity (L0 prose).** `code_id`: api-653 · `publisher`: API · `revision`: 3rd ed (2001) + 2003 Addendum (latest in `/mnt/ace/O&G-Standards/_catalog.json`); the publisher's currently-active edition is the **6th ed (2024)** — see *Edition history* below. Per spinout 2026-05-05 governance the wiki records publisher facts only; no clause text, formulas, or acceptance tables are reproduced.

## Scope

API 653 establishes the **in-service inspection, repair, alteration, and reconstruction** rules for **welded above-ground steel atmospheric storage tanks** that were originally designed to **API Std 650** or its predecessors in the **API 12-series** (API 12A, 12B, 12C, 12D, 12F field- and shop-welded tanks). It is the post-construction integrity counterpart to API 650 in the same way that API 510 is the in-service counterpart to ASME BPVC VIII for pressure vessels and API 570 is the in-service counterpart to ASME B31.3 for process piping.

Scope inclusions:

- Welded carbon-steel and stainless-steel atmospheric tanks storing hydrocarbons, chemicals, or water at near-atmospheric internal pressure (operating gauge pressure approximately at or below the API 650 design ceiling for atmospheric service).
- Tanks built to API 650 (current) or to the older API 12 series, whether riveted-then-rebuilt-as-welded, fully welded, or welded-with-bolted-roof.
- The full life-cycle decision pipeline: **routine in-service inspection → external inspection → internal inspection → engineering evaluation → repair / alteration / reconstruction → fitness-for-service escalation**.

Scope exclusions:

- **Pressurised vessels** above the API 650 atmospheric envelope route to **API 510** (Pressure Vessel Inspection Code) instead.
- **Underground storage tanks** are not in scope (those route to STI / EPA / state-level UST programs and to API RP 1631 / 1632 cathodic-protection guidance).
- **Low-pressure (non-atmospheric) tanks** designed to API Std 620 are out of scope; API 653 is API-650-tank-centric.
- New-tank design and construction is out of scope — that is API 650's domain. API 653 picks up at first commissioning and runs through the asset's retirement.

The covered population is large: the standard governs the in-service integrity programme for the order of **tens of thousands of refinery, terminal, midstream, and bulk-storage tanks worldwide**. In US refining and petrochemical operations API 653 compliance is invoked by **OSHA 29 CFR 1910.119 (PSM)** and **EPA 40 CFR 112 (SPCC)** programmes by reference, and by state-level above-ground storage tank regulations that adopt API 653 as the recognised consensus mechanical-integrity practice for atmospheric tanks.

## Key sections

API 653's structural body has been stable across the editions present in the catalog and the publicly documented editions since. The headings below track the long-running structure (section numbers and minor groupings shift across revisions but the substantive content is preserved):

- **Owner-user responsibilities.** The owner-user is accountable for tank mechanical integrity end-to-end: inspection programme governance, inspector qualification, inspection records, repair / alteration approval, and engineering evaluation of out-of-code findings. Authorised Inspectors performing API 653 inspections must hold the **API 653-ICP (Individual Certification Programme)** credential, the tank-side counterpart to the API 510-ICP (vessels) and 570-ICP (piping) certifications. The API 653-ICP examination is publicly administered by API ICP and is the load-bearing personnel-qualification gate for compliance-grade inspections.

- **Inspection types and intervals.** API 653 defines a layered inspection programme:
  - **Routine in-service inspection** — visual external surveillance performed by operations or maintenance personnel on a frequent (typically monthly) cadence; not an Authorised Inspector activity.
  - **External inspection** — performed by an Authorised Inspector at intervals **not exceeding five years** (or shorter where corrosion rate or service warrants), covering shell, foundation, anchorage, appurtenances, coatings, and visible bottom-edge / chime condition.
  - **Internal inspection** — performed by an Authorised Inspector at intervals driven by either prescriptive defaults or **risk-based inspection (RBI)** evaluation per **API RP 580 / 581**. The prescriptive ceiling is on the order of ten years for first internal and up to roughly twenty to twenty-five years between subsequent internals when supported by inspection history and corrosion-rate data; RBI can extend (or shorten) those intervals against documented risk targets.
  - **Hydrostatic settlement evaluation** — performed alongside internal inspections, comparing measured tank-floor and shell-base elevations against the tilt and differential-settlement acceptance envelopes defined in the standard.

- **Bottom-plate corrosion evaluation.** Tank bottoms see a damage profile distinct from shells: underside corrosion driven by sand-pad / soil-side electrochemistry, plus topside corrosion from product-water emulsion sitting at low spots. API 653 covers the evaluation toolkit:
  - **Sand-pad coupons** (driven coupon assemblies and external-side corrosion sampling) for underside-corrosion-rate measurement.
  - **Magnetic Flux Leakage (MFL)** floor-scanning for in-situ topside / underside metal-loss mapping during internal inspection.
  - **Acoustic emission** for in-service bottom-leak detection and pre-internal-inspection screening.
  - **Vacuum-box** and **tracer-gas leak detection** for confirming bottom-weld integrity.
  - Bottom **minimum-thickness retirement criteria** keyed to the corrosion mechanism and to whether secondary containment / a release-prevention barrier (per API RP 651 cathodic protection or API RP 652 lining) is in place.

- **Shell t<sub>min</sub> calculation.** Shell-course retirement thickness is computed using the **API 650 design equation framework** (the one-foot or variable-design-point methods, depending on tank size and joint efficiency) re-evaluated at the as-found product height, specific gravity, and joint efficiency — with API 653's own evaluation rules layered on top for **localised thinning**, **pitting density**, and **circumferential vs longitudinal stress orientation**. Where a shell course would fail the t<sub>min</sub> retirement check the standard offers the same three exit doors as API 510 / 570: **repair**, **operational re-rate** (reduce fill height to bring stress within the as-found wall capability), or **fitness-for-service** assessment per [api-std-579](api-std-579.md).

- **Repair and alteration categories.** API 653 distinguishes:
  - **Repairs** — work that restores the tank to its original design condition (shell-plate replacement, weld repair, coating restoration, bottom-plate patch or full replacement).
  - **Alterations** — changes from the original design (added nozzles, raised shell heights, re-rated capacity, jurisdiction-relevant nameplate changes) which trigger engineering review against the originally-applicable construction code (API 650 or the API 12 series) plus API 653's alteration-specific clauses.
  Both categories require a written repair / alteration procedure, qualified welders, NDE per the procedure, and post-work hydrostatic or vacuum-box leak testing as appropriate.

- **Reconstruction.** When a tank is dismantled and re-erected at the same or a new location, the reconstruction provisions of API 653 apply, layered on top of the **API 650 reconstruction provisions** (originally Appendix B of API 650, now the reconstruction sections of current API 650 editions). The reconstructed tank must be evaluated for material reuse acceptability, weld-joint integrity, and post-reconstruction hydrostatic test before return to service.

- **Inspector qualification.** API 653 inspections must be performed by a person holding the **API 653 Authorised Inspector** certification administered by API ICP. Re-certification is on a fixed interval and requires logged inspection experience plus continuing education / examination as defined by the ICP programme.

## How API 653 gates FFS

API 653 is the third member of the API in-service-inspection trilogy that gates **fitness-for-service** assessment for atmospheric storage tanks, parallel in posture to API 510 (pressure vessels) and API 570 (process piping):

- **Trigger condition.** When an inspection finding produces a **wall-loss measurement (shell, bottom, or roof) that drops the as-found thickness below the t<sub>min</sub> retirement threshold** computed per API 653's evaluation rules, the tank is in an out-of-code condition. The standard does not allow continued operation against the as-found thickness without a sanctioned exit.
- **Exit doors.** The owner-user has three sanctioned dispositions:
  1. **Repair** — restore the affected element to a thickness that passes t<sub>min</sub> (plate replacement, weld build-up where allowed, lining where allowed for bottom-side corrosion containment).
  2. **Operational re-rate** — reduce the maximum fill height so the as-found wall is sufficient at the reduced product head (a shell-stress-reduction route only, since bottom-plate hydrostatic loading does not scale with fill in the same way).
  3. **Fitness-for-Service** — perform a quantitative FFS assessment per **[api-std-579](api-std-579.md) / ASME FFS-1** that demonstrates the as-found thickness is acceptable at the original operating envelope despite failing the t<sub>min</sub> screening.
- **Damage-mechanism asymmetry vs sister codes.** The shell evaluation closely mirrors API 510 / 570 in posture (uniform-thinning Part 4 and locally-thin-area Part 5 of FFS-1 are the relevant routes), but the **tank-bottom evaluation is distinctive**: bottom-plate damage is dominated by widely scattered pitting and underside-corrosion patterns that do not have a direct analogue in pressurised-equipment service. FFS-1 Part 6 (pitting) and Part 5 (LTA) are both invoked, frequently in combination, for tank-bottom dispositions.
- **Code authority.** As with API 510 and API 570, FFS is **invoked under the authority of the inspection code, not as a free-standing decision**. A standalone FFS calculation without a documented API 653 inspection finding is not a sanctioned engineering deliverable. See [fitness-for-service](../concepts/fitness-for-service.md) for the broader gating concept.

## Cross-references

- [api-std-579](api-std-579.md) — **API 579-1 / ASME FFS-1**, joint API/ASME Fitness-for-Service standard. The downstream calculation engine when an API 653 inspection finds an out-of-code shell or bottom-plate condition. Storage-tank FFS most commonly routes through FFS-1 Parts 4 (general metal loss), 5 (LTA), and 6 (pitting); shell-distortion findings escalate to Part 8.
- **API Std 650** — *Welded Tanks for Oil Storage*. The original construction code for the population of tanks API 653 governs in service; API 653 reuses API 650's design equations for its t<sub>min</sub> retirement evaluations and its reconstruction provisions for dismantled-and-re-erected tanks.
- **API 510** — *Pressure Vessel Inspection Code: In-service Inspection, Rating, Repair, and Alteration*. Sibling in-service code for pressurised vessels; same code-gates-FFS posture, same Authorised-Inspector ICP credential family.
- **API 570** — *Piping Inspection Code: In-service Inspection, Rating, Repair, and Alteration of Piping Systems*. Sibling in-service code for process piping; completes the API in-service trilogy.
- **API RP 651** — *Cathodic Protection of Aboveground Petroleum Storage Tanks*. Recommended practice for sand-pad / soil-side cathodic-protection systems whose presence and condition feed API 653 bottom-plate evaluation rules.
- **API RP 652** — *Lining of Aboveground Petroleum Storage Tank Bottoms*. Recommended practice for internal-lining systems used as a bottom-side corrosion barrier; lining condition feeds API 653 internal-inspection findings and bottom-plate retirement decisions.
- **API 575** — *Inspection Practices for Atmospheric and Low-Pressure Storage Tanks*. Inspection-practice companion document; describes inspection techniques, equipment, and damage-mechanism recognition supporting the formal inspection programme that API 653 codifies.
- **API RP 580 / 581** — *Risk-Based Inspection / RBI Technology*. The RBI methodology that API 653 invokes to extend or shorten internal-inspection intervals from prescriptive defaults.
- **ASME BPVC IX** — Welding and brazing qualifications used for repair / alteration weld procedures and welder qualification under API 653. See [asme-bpvc-ix](asme-bpvc-ix.md).
- **AWS D1.1** — Structural welding code referenced for steel-construction welding outside ASME BPVC IX scope where applicable. See [aws-d1-1](aws-d1-1.md).
- [fitness-for-service](../concepts/fitness-for-service.md) — concept page for the FFS gating pattern shared across API 510 / 570 / 653.
