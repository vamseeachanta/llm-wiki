# API 570 — Piping Inspection Code: In-Service Inspection, Rating, Repair, and Alteration of Piping Systems

> **code_id:** `api-570` &nbsp;·&nbsp; **publisher:** API &nbsp;·&nbsp; **revision:** catalog holds 1998 (ICO + Addenda 1–3), 2001, and 2006 (Addenda) editions; current published edition is 3rd ed (2016) with subsequent addenda

## Scope

API 570 is the owner-user inspection code governing **in-service inspection,
rating, repair, and alteration of metallic piping systems** in process and
refining plants. It applies to piping designed and constructed under
[asme-b31-3](asme-b31-3.md) (Process Piping) — and, by analogy, sister B31 piping codes —
once the system has been placed in service. API 570 is the piping companion
to **API 510** (pressure vessels) and **API 653** (atmospheric storage tanks):
together the three codes form the API in-service inspection trilogy that
gates fitness-for-service decisions across refinery and petrochemical
asset bases.

The code covers piping that handles process fluids, hydrocarbons, and other
flammable / hazardous materials; utility piping (e.g., water, low-pressure
steam, plant air) is generally outside primary scope but is treated under
the lowest piping class. API 570 is **not** a design code — new construction
remains under B31.3 — but it prescribes the inspection, condition-monitoring,
remaining-life, repair-and-alteration, and re-rating workflows that govern
the asset for the rest of its operational life.

## Key sections

API 570 is organised around the operational lifecycle of in-service piping.
The headings below summarise the standard's structural backbone — clause
text, formulas, and tabulated criteria are **not reproduced** per
metadata-only governance.

- **Owner-user responsibilities.** The owner-user is accountable for
  inspection programme execution, written inspection plan, qualification
  of inspectors and examiners, integrity-operating-window (IOW)
  monitoring, and the recordkeeping that makes inspection results
  defensible. Responsibility cannot be delegated to a third-party
  contractor; the owner-user remains the accountable party.

- **Piping classification.** Piping circuits are categorised by
  consequence-of-failure into five classes — broadly, **Class 1**
  (highest-consequence: flammable services that auto-refrigerate or
  flash on release; high-toxicity services such as anhydrous HF, H₂S
  in significant concentration; severe-cyclic services), through
  **Class 2** and **Class 3** (general process services with
  decreasing consequence), down to **Class 4** and **Class 5**
  (lowest-consequence services such as water, low-pressure steam,
  utility air). Class drives the maximum permitted inspection
  interval and the minimum required CML coverage. Owner-users may
  use a more granular classification scheme so long as the API 570
  classes are preserved as a floor.

- **Inspection intervals.** The maximum interval between thickness
  inspections is the lesser of **half the remaining life** of the
  circuit OR a class-dependent ceiling (typically measured in years).
  Internal/external visual inspection follows a parallel half-life /
  class-ceiling rule. **Risk-based inspection (RBI)** per
  [api-rp-580](api-rp-580.md) / [api-rp-581](api-rp-581.md) is an explicitly permitted
  alternative basis for setting intervals, provided the RBI study is
  documented and re-evaluated at code-mandated frequency.

- **Condition Monitoring Locations (CMLs).** CMLs (formerly TMLs —
  Thickness Measurement Locations) are designated points on a piping
  circuit where wall thickness is measured on a recurring schedule
  to track corrosion. CML selection prioritises:
  - **High-corrosion zones** (predicted hot-spots from process
    corrosion modelling),
  - **Dead legs** (stagnant flow regions with elevated localised
    corrosion risk),
  - **Injection points** (chemical injection, water washing,
    additive injection — turbulent local mixing accelerates
    erosion-corrosion downstream),
  - **Mix points**, **soil-air interfaces**, **insulation-jacketing
    transitions** (CUI risk), and **velocity-elevated zones**
    (elbows, reducers, control valves).
  CML count and spacing are class-driven, with Class 1 carrying the
  densest CML coverage and shortest re-measurement interval.

- **Thickness measurement methods.** Ultrasonic thickness (UT) is the
  default, supplemented by radiographic profile (RT), automated
  ultrasonic (AUT) / phased-array (PAUT), guided-wave UT for long
  runs, EMAT for high-temperature service, and pulsed eddy current
  for through-insulation screening. Method selection is a function of
  pipe geometry, accessibility, insulation status, and required
  measurement uncertainty.

- **Corrosion-rate calculation.** Two rates are tracked:
  - **Long-term corrosion rate (LTCR)** = (initial or commissioning
    thickness − current thickness) / time-in-service,
  - **Short-term corrosion rate (STCR)** = (previous-inspection
    thickness − current thickness) / time-between-inspections.
  Remaining life is computed from current thickness less retiring
  (minimum allowable) thickness, divided by the **higher** of LTCR
  / STCR. STCR > LTCR is a standing red flag indicating accelerated
  recent corrosion (process upset, IOW excursion, new mechanism).

- **MAWP recalculation.** When measured thickness drops below
  the as-designed thickness but remains above the retiring
  thickness, the maximum allowable working pressure (MAWP) is
  recalculated from the remaining wall using the original
  construction code's (B31.3 etc.) pressure-design equation.
  Re-rating to a lower MAWP is permitted if it preserves design
  margin; otherwise the circuit must be repaired, replaced, or
  routed to fitness-for-service per below.

- **Repair categories and welding requirements.** Repairs and
  alterations are categorised by extent and consequence (e.g.,
  full-encirclement welded sleeve, fillet-welded patch, on-stream
  hot-tap, weld build-up). All welding work requires qualified
  procedures (**WPS** — Welding Procedure Specification, supported
  by **PQR** — Procedure Qualification Record) per [asme-bpvc-ix](asme-bpvc-ix.md)
  and the relevant construction code. **Hot work in service** routes
  through [api-rp-2201](api-rp-2201.md) (hot tapping). Temporary repairs have
  defined service-life ceilings before permanent repair is required.

- **Inspector qualifications.** Inspectors performing API 570
  inspections must be certified to the **API 570 Inspector
  Certification Programme (API-570-ICP)**. NDE examiners must hold
  ASNT SNT-TC-1A (or equivalent — CP-189, ACCP) qualification at
  Level II minimum for the methods they perform. The inspector and
  examiner roles are distinct: examiners take the data; inspectors
  interpret it against code criteria.

## How API 570 gates fitness-for-service

API 570 mirrors the gating pattern of API 510 and API 653: **the inspection
code identifies an out-of-code condition, and fitness-for-service is the
sanctioned engineering path forward.** Specifically, when piping inspection
finds:

- **Wall loss exceeding code retiring thickness** (i.e., measured thickness
  below the minimum required by the construction code's pressure-design
  equation at current operating conditions), or
- **Crack-like flaws** (planar defects detected by RT, UT, PAUT, ACFM, or
  surface NDE) above the inspection-code acceptance threshold, or
- **Distortion / dents / mechanical damage** beyond geometric tolerance,

the owner-user has three routes:

1. **Repair** to restore the original code-compliant condition,
2. **Replace** the affected component or circuit segment,
3. **Invoke [api-std-579](api-std-579.md) (API 579-1 / ASME FFS-1)** to demonstrate
   that the as-found component is fit for continued service under
   defined operating envelopes, with a remaining-life estimate and
   re-inspection interval.

API 570 explicitly delegates to API 579-1 for the FFS calculation; the
inspection code defines the trigger, the FFS standard provides the
quantitative assessment. See [fitness-for-service](../concepts/fitness-for-service.md) for the cross-code
context of this gating pattern. **A standalone FFS calculation without
a documented API 570 inspection finding is not a sanctioned engineering
deliverable** — the inspection code's authority is what makes the FFS
result actionable.

## Cross-references

- **[asme-b31-3](asme-b31-3.md)** — *Process Piping* — the new-construction design
  code that API 570 inspects against. Retiring-thickness, MAWP, and
  re-rating calculations all route back through B31.3 pressure-design
  equations.
- **API 510** — *Pressure Vessel Inspection Code* (sibling). Same
  owner-user / inspector / FFS-gating pattern applied to pressure
  vessels; cross-trained inspector workforce in practice.
- **API 653** — *Tank Inspection, Repair, Alteration, and
  Reconstruction* (sibling). Same pattern applied to atmospheric and
  low-pressure storage tanks.
- **[api-std-579](api-std-579.md)** — *API 579-1 / ASME FFS-1, Fitness-for-Service*.
  The FFS standard invoked when API 570 inspection finds an
  out-of-code condition.
- **[api-rp-580](api-rp-580.md)** — *Risk-Based Inspection*. Methodology framework
  for risk-based interval-setting alternative.
- **[api-rp-581](api-rp-581.md)** — *Risk-Based Inspection Methodology*. Quantitative
  RBI consequence/likelihood calculation that consumes API 570
  thickness and FFS remaining-life inputs.
- **[api-rp-574](api-rp-574.md)** — *Inspection Practices for Piping System
  Components*. Inspector-grade practice guide companion to API 570.
- **[api-rp-577](api-rp-577.md)** — *Welding Inspection and Metallurgy*. Practice
  guide for the welding-inspection portion of API 570 repairs.
- **[api-rp-578](api-rp-578.md)** — *Material Verification Program for New and
  Existing Alloy Piping Systems*. PMI (positive material identification)
  practice for alloy-piping receipt and in-service verification.
- **[asme-bpvc-ix](asme-bpvc-ix.md)** — *BPVC Section IX, Welding and Brazing
  Qualifications*. Source for WPS / PQR / welder-performance
  qualification rules invoked by API 570 repair clauses.
- **[api-rp-2201](api-rp-2201.md)** — *Procedures for Welding or Hot Tapping on
  Equipment in Service*. Hot-work-in-service authority cited from
  API 570 repair sections.
- **[fitness-for-service](../concepts/fitness-for-service.md)** — concept-page consumer of this
  standards page. FFS gating pattern documented across API 510 / 570 /
  653 routes through this page for piping.
