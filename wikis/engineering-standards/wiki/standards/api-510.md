# API 510 — Pressure Vessel Inspection Code: In-Service Inspection, Rating, Repair, and Alteration

> **code_id:** `api-510` &nbsp;·&nbsp; **publisher:** API &nbsp;·&nbsp; **revision:** 9th ed (catalog latest); 11th ed (2022) is the current published edition

## Scope

API 510 is the in-service inspection, rating, repair, and alteration code for
pressure vessels and the pressure-relieving devices that protect them, in
process plants where vessels are constructed (or were originally constructed)
to ASME *Boiler and Pressure Vessel Code* Section VIII Division 1, Division 2,
or Division 3, or to comparable predecessor codes. It applies after a vessel
has been placed in service: it does **not** govern new-build construction
(that is ASME BPVC VIII's territory) and it does **not** apply to piping
systems (which fall under [api-std-570](api-std-570.md)) or to atmospheric / low-pressure
storage tanks (which fall under [api-std-653](api-std-653.md)). The code defines the
owner-user inspection programme, inspection-interval setting (table-based or
risk-based per [api-rp-580](api-rp-580.md) / [api-rp-581](api-rp-581.md)), inspector qualification (the
API 510 Individual Certification Program — ICP), inspection methods and
techniques (visual, UT, MT, PT, RT, AET — calling the practice documents
[api-rp-572](api-rp-572.md) and [api-rp-577](api-rp-577.md)), corrosion-rate analysis (long-term and
short-term), the determination of minimum required thickness `t_min` from the
applicable ASME construction-code calculation, and the technical
requirements for repairs and alterations (including the role of an *Approved
Repair Organization* and post-weld heat-treatment / *Alternative Local Heat*
options). When inspection finds an out-of-tolerance condition — metal loss
below `t_min`, crack-like flaws, dents, blisters, or distortion — API 510
governs the disposition decision: repair to original code, re-rate, or
invoke [api-std-579](api-std-579.md) / ASME FFS-1 to demonstrate fitness for continued
service at the as-found condition.

## Key sections

API 510 is organised around the owner-user programme rather than a strict
clause-numbering inheritance from ASME BPVC. The functional structure below
is consistent across editions, with section numbers shifting between the 9th,
10th, and 11th editions.

- **Owner-user inspection programme.** The owner-user is the responsible
  party for the inspection programme. Required programme elements include
  written procedures, inspector and examiner qualifications, equipment
  records (manufacturer's data report, repair history, inspection history),
  pressure-relieving-device testing schedules, and documented quality
  assurance.
- **Inspection intervals.** Two routes are sanctioned:
  - **Table-based.** Maximum interval is the lesser of one-half the
    remaining-life (computed from corrosion rate and `t_actual − t_min`) or
    a code-tabulated cap (commonly 10 years for external visual /
    on-stream, 5–10 years for internal / UT, edition-dependent).
  - **Risk-based (RBI).** Per [api-rp-580](api-rp-580.md) (RBI methodology) and
    [api-rp-581](api-rp-581.md) (RBI quantitative implementation). RBI permits longer
    intervals when probability-of-failure × consequence-of-failure
    justifies it, conditional on a documented RBI assessment that meets
    the owner-user programme requirements.
- **Inspector qualifications.** Inspectors must hold the API 510
  Individual Certification (the *API 510 ICP*) for authorised inspection
  signoff. Examiners (NDE technicians who perform but do not authorise
  the examination) are qualified per the applicable NDE standard
  (ASNT SNT-TC-1A, CP-189, or equivalent).
- **Inspection methods.** Visual external and internal, ultrasonic
  thickness (UT), wet fluorescent magnetic particle (WFMT), liquid
  penetrant (PT), radiographic (RT), acoustic emission testing (AET).
  Method selection is governed by the damage-mechanism profile and is
  developed per [api-rp-572](api-rp-572.md) (inspection practices for pressure
  vessels) and [api-rp-577](api-rp-577.md) (welding inspection and metallurgy), with
  weld-quality and welder-qualification cross-references to ASME BPVC
  Section IX.
- **Corrosion-rate analysis.** Both **long-term** (LT, baseline-to-current
  trend) and **short-term** (ST, recent inspection cycles) rates are
  computed from thickness measurements at established CMLs (Condition
  Monitoring Locations). The governing rate for remaining-life
  computation is the higher of LT and ST.
- **Thickness limits.** Minimum required thickness `t_min` is computed
  from the applicable ASME construction-code formula (BPVC VIII Div 1
  UG-27 / UG-32 for shells and heads; BPVC VIII Div 2 Part 4 design-by-
  rule formulas; BPVC VIII Div 3 for high-pressure vessels). The
  measured `t_actual` minus the future corrosion allowance over the next
  inspection interval determines remaining life.
- **Repair and alteration requirements.** Repairs and alterations must be
  performed by an *Approved Repair Organization* operating under the
  owner-user's quality programme, using procedures, welders, and NDE
  qualified to ASME BPVC Section IX (welding) and the construction code
  of record. Post-weld heat treatment is per the construction code; an
  *Alternative Local Heat-treatment* option (sometimes labelled ALH or
  *Local PWHT*) is sanctioned for in-situ repairs where full vessel
  PWHT is impractical, subject to additional NDE.
- **Pressure-relieving devices.** PRDs are inspected, tested, and
  re-set on a schedule defined by the code; inspection practice is
  developed in [api-rp-576](api-rp-576.md).
- **FFS invocation.** When `t_actual < t_min`, when crack-like flaws
  are detected, or when geometric out-of-tolerance is found, API 510
  delegates the disposition decision to [api-std-579](api-std-579.md) / ASME FFS-1.

## How API 510 gates FFS

When inspection finds a non-conforming condition, the owner-user has three
documented routes under API 510:

1. **Repair to the original construction code.** Restore the vessel to a
   condition that satisfies the applicable ASME BPVC VIII division using
   an Approved Repair Organization. The repaired vessel is re-inspected
   and returned to service under the existing rating.
2. **Re-rate.** Reduce the maximum allowable working pressure (MAWP) or
   maximum allowable working temperature (MAWT) so that the as-found
   thickness or geometry satisfies the applicable construction-code
   formula at the reduced rating. The re-rating must be documented and
   the vessel re-stamped or re-marked per the owner-user programme.
3. **Invoke FFS per [api-std-579](api-std-579.md).** Demonstrate that the as-found
   condition is fit for continued service at the existing rating using
   the API 579-1 / ASME FFS-1 Level 1 / 2 / 3 assessment methodology
   appropriate to the damage mechanism (Part 4 general metal loss,
   Part 5 local metal loss, Part 6 pitting, Part 7 blisters /
   laminations, Part 8 distortion, Part 9 crack-like flaws, etc.). The
   FFS output (acceptance verdict, remaining-strength factor, remaining
   life) is the basis for the run / monitor / re-inspect plan. FFS is
   not a free-standing decision; it is invoked under the authority of
   API 510, and the assessment record is filed with the vessel
   inspection history.

The decision among these three routes — typically governed by §7 of the
code in editions 9–11 — is the owner-user's, made on technical and
economic grounds with sign-off from the API 510 ICP-certified inspector
and (for FFS) a qualified engineer. The **gating function of API 510 is
load-bearing**: a standalone FFS calculation without a documented
inspection finding under API 510 (or the analogous piping / tank gates
under API 570 / API 653) is not a sanctioned engineering deliverable.
This is the upstream constraint recorded in the
[fitness-for-service](../concepts/fitness-for-service.md) concept page.

## Cross-references

- **ASME BPVC Section VIII Div 1.** Design-by-rule construction code for
  pressure vessels — the source of the `t_min` formulas (UG-27 shells,
  UG-32 heads, etc.) that API 510 invokes for in-service thickness
  evaluation. See [asme-bpvc-viii-1](asme-bpvc-viii-1.md).
- **ASME BPVC Section VIII Div 2.** Design-by-analysis alternative-rules
  construction code; source of design-by-analysis stress classification
  used when re-rating or FFS-1 Level 3 assessments are performed. See
  [asme-bpvc-viii-2](asme-bpvc-viii-2.md).
- **ASME BPVC Section IX.** Welding and brazing qualifications — applies
  to welder, welding-procedure, and NDE-procedure qualification for
  API 510 repairs and alterations. See [asme-bpvc-ix](asme-bpvc-ix.md).
- **API 579-1 / ASME FFS-1.** The fitness-for-service standard invoked
  when an out-of-code condition is found during API 510 inspection.
  See [api-std-579](api-std-579.md).
- **API RP 580 / API RP 581.** Risk-based inspection methodology
  (RP 580 — qualitative / framework; RP 581 — quantitative
  implementation). Provide the alternative interval-setting route to
  API 510's table-based intervals.
- **API RP 572 — Inspection Practices for Pressure Vessels.** Practice
  document on damage-mechanism recognition and inspection technique
  selection for vessels; the applied-practice companion to API 510's
  programmatic requirements.
- **API RP 577 — Welding Inspection and Metallurgy.** Practice document
  on weld-inspection metrology, defect recognition, and welder-
  qualification monitoring; applied to API 510 repairs.
- **API RP 576 — Inspection of Pressure-relieving Devices.** Companion
  practice document for PRD inspection / testing under API 510's
  pressure-relieving-device programme.
- **API 570 — Piping Inspection Code.** Sibling in-service inspection
  code for process piping; analogous owner-user programme structure;
  invokes API 579-1 for piping FFS. See [api-std-570](api-std-570.md).
- **API 653 — Tank Inspection, Repair, Alteration, and Reconstruction.**
  Sibling in-service inspection code for atmospheric / low-pressure
  storage tanks; analogous owner-user programme structure; invokes
  API 579-1 for tank FFS. See [api-std-653](api-std-653.md).
- **API 510 ICP — Individual Certification Program.** API's
  inspector-certification programme. Authorised-inspector signoff on
  vessel inspection records under API 510 requires an active 510-ICP
  credential. The ICP scheme is administered by API ICP separately
  from the published code; certification and re-certification cycles
  follow the ICP body's requirements (typically 3-year cycle with
  CEU / re-exam paths).
- **Fitness-for-Service concept.** [fitness-for-service](../concepts/fitness-for-service.md) — the
  concept page that identifies API 510 (with API 570 and API 653) as
  the upstream gate on FFS invocation.
