# Compound Engineering

The core methodology running the ACE Engineer workspace-hub ecosystem. Engineering work produces tools, tools produce better work, better work produces better tools. The flywheel.

## The Five Lessons

1. **[[enforcement-over-instruction]]** -- text-based rules get bypassed; technical gates don't
2. **[[orchestrator-worker-separation]]** -- coordinator + isolated workers beats one agent with everything
3. **[[compound-learning-loop]]** -- skills improve from real work, not upfront design
4. **[[multi-agent-parity]]** -- shared knowledge base eliminates redundant discovery
5. **[[three-agent-cross-review]]** -- independent reviewers for plans AND artifacts

## How It Works

```
Real engineering work
    → Tools and scripts created
        → More work done faster
            → Better tools refined
                → Compound effect over time
```

## Evidence

- 691+ skills across multiple agents (Claude, Codex, Gemini, Hermes)
- Operational since 2025-Q2, continuously improving
- Multiple machines coordinated via git-tracked configuration

## Key Insight

This is not designed methodology -- it emerged from doing real engineering work. The patterns were discovered through practice, not planned upfront.
