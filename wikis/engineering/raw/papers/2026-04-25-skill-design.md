# Research: skill-design — 2026-04-25

## Key Findings

- **Agent Skills as open standard (SKILL.md) enables cross-vendor interoperability — adopted by 20+ AI coding agents (Claude Code, Cursor, Codex, etc.) as of April 2026. Progressive disclosure pattern (lightweight metadata upfront, full instructions loaded only when relevant) enables agents to manage 100+ skills without context degradation. Naming convention: gerund form (verb + -ing) for clarity. Skills contain two types: reference content (conventions, patterns, domain knowledge — runs inline) and task content (step-by-step instructions for specific actions — invoked directly via /skill-name).** Skills vs. MCP distinction: Skills are knowledge/interoperability layer; MCP is tool/connectivity layer. In multi-agent architectures, shared SKILL.md format allows agents from different providers (Claude, OpenAI, Google) to coordinate without custom integration. workspace-hub currently uses inline micro-skills (`.claude/skills/workspace-hub/stages/`) for per-stage rules; April 2026 landscape suggests adopting formal SKILL.md frontmatter (metadata + task instructions) for public-facing skills enabling cross-team delegation and skill registries.

- **Progressive disclosure as context-window efficiency pattern (February–April 2026 research): organizing agent instructions into three tiers (lightweight metadata → task-relevant instructions → supporting docs/examples) enables scaling from ~25 skills (no disclosure) to 100+ skills (aggressive disclosure) without context degradation. Practical implementation: skill metadata (name, description, invocation trigger) ≤ 200 tokens; full instructions loaded only after agent determines relevance. Key insight — agents scoring low-relevance skills skip loading, so context footprint is determined by task complexity, not skill catalog size.** workspace-hub's current memory structure (CLAUDE.md 20-line max, AGENTS.md compact, KNOWLEDGE.md modular topics) aligns with disclosure tier 1 (metadata). Opportunity: formalize tier 2/3 (task-relevant instructions, supporting examples) in skill organization to enable 50+ cross-repo skills without current context bloat.

- **Agent evaluation framework maturity (2026): Evals standardized as prompt → captured trace → checks → score → historical comparison. Regression testing gates block deployments reducing quality below thresholds. Canary deployment pattern (5% traffic for 24-48h, auto-incident-to-test conversion) now paired with SkillLearnBench benchmarking of skill generation methods. workspace-hub Phase 5 nightly researchers + Phase 7 orchestrator are ideal canary candidates — production traces feed directly into regression test suite.** Continuous assessment schedule: weekly health checks (latency, cost, error rates), monthly goal fulfillment deep dives, quarterly comprehensive regression. workspace-hub currently has ~90.5% test coverage (v1.0 Phase 1–6); April 2026 research validates that threshold gating (90%+ enforcement via pytest-cov/coverage.py 7.13.5) is now production-standard.

- **Multi-agent orchestration as infrastructure (2026): Explicit task boundaries, isolated execution, evidence-based merges. Claude Code Agent Teams (peer-to-peer mailbox coordination) enables role-specific delegation with shared context/memory. Skill registries (unsigned, signed, or audited distribution) enable dynamic discovery at runtime. 40% of enterprise applications will feature task-specific agents by 2026 (up from <5% in 2025).** workspace-hub Phase 7 adopts Agent Teams mailbox pattern; prior research (2026-04-22) corrected assumption from hierarchical RPC to peer-to-peer. Skill registry opportunity: `.claude/skills/registry.yaml` (skill metadata + invocation triggers + ownership) enables Phase 7 agents to self-serve skill discovery without hardcoded imports. Delegation boundary clarity (solver agent owns OrcFxAPI access only, orchestrator owns task dispatch) reduces operational risk.

- **CLAUDE.md/AGENTS.md best practices (2026): Keep harness files ≤ 20 lines (excess migrates to skills/docs). CLAUDE.md contains only universally applicable instructions; stage-specific rules live in micro-skills. Reference documents listed in CLAUDE.md with brief descriptions; ask Claude to read selectively. Command → Agent → Skill architecture separates orchestration (command) from provider choice (agent) from capability packaging (skill).** workspace-hub's `.claude/rules/` enforcement (check-no-abs-paths.sh, check-harness-file-size.sh) validates 20-line constraint at pre-commit; patterns.md documents migration to script-level enforcement (Level 0→3 spectrum). April 2026 research validates this approach: workspace-hub is ahead of community baseline on harness discipline.

`★ Insight ─────────────────────────────────────`

**Three skill architecture inflection points enable Phase 7→Phase 999.4 progression:**

**1. SKILL.md Open Standard + Progressive Disclosure = Interop + Scale**

April 2026 landscape validates Agent Skills (SKILL.md) as cross-vendor standard (20+ vendors, 10K+ MCP servers). Progressive disclosure (3-tier: metadata ≤200 tokens, task instructions on relevance, supporting docs optional) enables workspace-hub to scale from ~50 current skills to 100+ without context degradation. **Immediate value:** Phase 7 Agent Teams agents (Claude Code + Codex) recognize identical skill metadata — no custom parsing per vendor. **Deferred value:** Phase 1.1 client engagements can export workspace-hub skills (e.g., DNV-RP-C203 fatigue calculator) for partner integration (consulting-value IP).

**2. Skill Registries as Runtime Discovery Mechanism**

Current Phase 7 orchestrator hardcodes skill imports (e.g., `from .skills import orcaflex_solver`). Registry-based discovery (`.claude/skills/registry.yaml` queried at runtime) decouples skill evolution from orchestrator code. Phase 7 agents self-serve skill discovery; Phase 999.4 agents extend this pattern to cross-agent skill invocation (solver agent → evaluator agent, feedback loop). Enables multi-agent autonomous systems without centralized hardcoding.

**3. Evaluation Framework + Regression Gates = Quality Continuity**

Phase 5 nightly researchers + Phase 7 smoke tests are canary environments. Production traces (execution traces, output artifacts) feed directly into regression test suites. Incident-to-test conversion (bad decision by Phase 5 researcher → test case preventing recurrence) turns operational issues into permanent quality improvements. Phase 7 regression gates (pass-rate > 99.5%, calculation error < 1e-6) enforce shipping discipline. Phase 999.4 autoresearch iterates within these gates (improvements accepted only if not degrading existing quality).

Together: SKILL.md interop + progressive disclosure + registries + evaluation gates create a **platform for multi-agent automation at scale**. Phase 7 proves it with Agent Teams orchestration. Phase 999.4 generalizes to research automation.
