No breaking changes to current SSE transport (Phase 7 uses SSE, ships mid-May before spec release). ([The 2026 MCP Roadmap](https://blog.modelcontextprotocol.io/posts/2026-mcp-roadmap/), [Exploring the Future of MCP Transports](https://blog.modelcontextprotocol.io/posts/2025-12-19-mcp-transport-future/))

- **Codex CLI April 2026: Filesystem permissions deny-read glob policies (granular per-module isolation), sandbox network access granular controls (package-managers-only vs. full internet vs. allowlist), permission profiles round-trip across TUI/user-turns/MCP sandbox state/shell escalation/app-server APIs, native Windows AppContainer-based sandbox (restricts filesystem writes, blocks network by default). No May 2026 updates documented yet (future month).** Deny-read globs + Windows AppContainer represent production-grade security hardening for Phase 7.1 multi-machine expansion (if licensed-win-2 joins solver pool, AppContainer-enforced network isolation prevents exfiltration of OrcFxAPI results). ([Codex Sandbox Concepts](https://developers.openai.com/codex/concepts/sandboxing), [Codex Changelog](https://developers.openai.com/codex/changelog))

Mailbox system enables direct peer-to-peer messaging (any teammate messages any other, not hierarchical reporting). Shared task list + independent context windows (1M-token per agent, measured "substantially more" overhead than single session but acceptable for independent parallel work). Feature shipped February 2026 (experimental), moving toward production status based on validation runs.** Mailbox validation at Anthropic internal scale (54% review coverage on thousands of PRs) confirms Phase 7 architecture is production-proven. Prior research assumption (hierarchical RPC) corrected; actual implementation is peer-to-peer mailbox. ([Claude Code Agent Teams Documentation](https://code.claude.com/docs/en/agent-teams), [Claude Code Agent Teams Setup & Usage Guide 2026](https://claudefa.st/blog/guide/agents/agent-teams))
