## Key Strategic Insights for Phase 7 Execution

### 1. **Agent Teams Mailbox Pattern Corrects Prior Architecture Assumption**

Prior research (2026-04-15) assumed hierarchical A2A delegation (orchestrator → solver subagents with RPC). **Actual implementation:** peer-to-peer via mailbox system — orchestrator assigns tasks to shared list, solver agents consume independently, post results to mailbox. This is **simpler and more reliable** (single source of truth: mailbox state, no polling timeouts on agent readiness). **Action:** Redesign Phase 7 orchestrator from hierarchical RPC model to mailbox polling. Implement task schema with dependency tracking (task C waits for [A, B] completion). Measure overhead on 2–3 smoke tests before scaling to 5–6 parallel solvers.

### 2. **Auto Mode (Safe Action Classifier) Enables Remote Execution Safety**

Auto Mode (Sonnet 4.6-based classification) reviews every action before execution, auto-approving safe actions and blocking risky ones. For Phase 7 licensed-win-1 remote execution, this creates a critical safety gate: **orchestrator can't accidentally trigger wrong load case, overwrite fixture, or cause data loss.** < 5% false positive rate is target. **Action:** Integrate Auto Mode into Phase 7 CI/CD gate immediately; measure false positive rate on 5–10 test runs; if acceptable, enable whitelisting for orchestration actions.

### 3. **Ultraplan (Cloud Planning) Addresses Phase 7 Transparency Requirement**

Current Phase 7 PLAN.md generation is local; Ultraplan enables cloud-based planning (draft orchestrator task list in cloud, team reviews/comments in browser, pulls back for local execution). If Ultraplan reaches beta by May 15, Phase 7 can adopt it as **transparency mechanism** — stakeholder reviews, decision traceability, team alignment captured in cloud artifact before execution begins. **Timeline:** Ultraplan is early preview; monitor for beta; if shipped on schedule, integrate into Phase 7 planning gate.

### 4. **GSD v1.36 Knowledge Graph Unlocks Phase 7.1 Intelligent Failure Diagnosis**

/gsd-graphify (knowledge graph for planning agents) directly enables Phase 7.1 feature: "intelligent solver state inspection" — query graph to find which modules consume OrcFxAPI, trace dependency changes on version updates. Pattern mapper (pre-configured agent) automatically identifies coupled modules in failure triage. **Action:** Upgrade GSD to v1.36 immediately; design Phase 7.1 graph queries; defer Phase 7.1 implementation to post-May planning.

### 5. **MCP Stateless HTTP (June 2026) Unblocks Phase 999.4+ Multi-Solver Federation**

Phase 7 uses current SSE transport (stable, no breaking changes). June 2026 spec release brings stateless HTTP (load balancing across multiple solver machines without session affinity). **Timeline:** If June 2026 publication holds, Phase 7.1+ can leverage for licensed-win-1 + licensed-win-2 + future machine load balancing. If delayed, Phase 7.1 uses custom workarounds (MCP doesn't regress).
